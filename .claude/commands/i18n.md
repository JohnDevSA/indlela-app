# Add Translations

Add translation strings across all 6 South African languages supported by Indlela.

## Instructions

When the user runs `/i18n <key> <english_text>`, add the translation to all locale files.

## Supported Languages

| Code | Language | Native Name |
|------|----------|-------------|
| en | English | English |
| zu | isiZulu | isiZulu |
| xh | isiXhosa | isiXhosa |
| af | Afrikaans | Afrikaans |
| st | Sesotho | Sesotho |
| tn | Setswana | Setswana |

## File Locations

- `locales/en.json`
- `locales/zu.json`
- `locales/xh.json`
- `locales/af.json`
- `locales/st.json`
- `locales/tn.json`

## Key Format

Use nested keys with dot notation:
- `booking.status.pending`
- `provider.profile.bio`
- `common.loading`

## Translation Guidelines

1. **Context Matters**: Consider the UI context for accurate translation
2. **Formal/Informal**: Use respectful forms appropriate for a service marketplace
3. **Gender Neutral**: Where possible, use gender-neutral terms
4. **Cultural Sensitivity**: Be mindful of cultural nuances in townships
5. **Length**: Some languages expand significantly - keep translations concise

## Example
